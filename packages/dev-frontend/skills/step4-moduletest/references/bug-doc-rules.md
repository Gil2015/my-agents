# 缺陷文档填写规则

`.ai/missions/{module}/bugDocs/bug.md` 在问题收集阶段的填写与更新规则。主模板以 `../../references/bug-doc-template.md` 为准，本文件补充收口阶段的字段约束和进度摘要规则。

## 顶部摘要填写规则

| 字段 | 填写方式 | 说明 |
|------|---------|------|
| 模块名 | 与实际模块目录一致 | 不写页面标题或口语简称 |
| 当前结论 | 1 句总结本轮发现 | 例如“本轮新增 2 个 OPEN 缺陷，先处理删除和空值链路” |
| 修复进度 | `OPEN x / FIXING y / FIXED z / BLOCKED n / WONT_FIX m` | 必须由当前条目状态统计得出 |
| 本轮来源 | 列出真实来源组合 | 例如 `USER_INPUT + bugDocSources + reqDocs/req.md audit` |
| 优先处理 | `BUG-*` 列表或 `无` | 列出建议优先修复的条目 |

## `BUG-*` 编号规则

- 每个缺陷使用唯一 `BUG-*` 编号
- 同一文档内编号递增，不复用旧编号
- 已有缺陷保留原编号，新增缺陷使用下一个可用编号
- 收集阶段不要因为“看起来重复”就轻易删除旧条目；先确认是否同根因

## 单条缺陷字段规则

| 字段 | 填写方式 | 说明 |
|------|---------|------|
| 标题 | 一句话描述问题 | 要能直接看出异常点 |
| 严重级别 | `S0` / `S1` / `S2` / `S3` | 与真实影响匹配 |
| 来源 | `用户反馈` / `bugDocSources` / `AI审查` 等 | 可写组合来源 |
| 来源文件 | 真实路径或 `无` | 来自 `bugDocSources` 时尽量填写 |
| 状态 | `OPEN` / `BLOCKED` | 收集阶段通常只允许这两种 |
| 关联需求 | `REQ-*` / `AC-*` / `USER_INPUT` / 留空 | 没有真实编号时不要伪造 |
| 关联代码 | 模块内真实文件路径或链路描述 | 尽量让 step5 快速定位 |
| 关联接口 | `API-*` / 接口标识 / `无` | 没有接口依赖时写 `无` |
| 复现环境 | 浏览器、账号、环境或“代码审查，未执行页面” | 方便后续复核 |
| 剩余风险 | 1 句说明 | 写可能受同一路径影响的相邻风险 |
| 是否关闭 | `否` | 收集阶段不要提前关闭问题 |

## 内容填写要求

### `问题描述`

- 简要描述症状和用户感知
- 适合保留用户口述原意或来源文件的原始问题摘要

### `复现步骤`

- 尽量写到别人可以继续验证的程度
- 静态审查发现的问题，也要写出最小触发路径
- 如果确实未执行页面，可在步骤里写明“待进入修复前补验证”

### `期望结果`

- 必须能和 `req.md`、显式用户描述或接口契约对上
- 不写主观话术，例如“体验良好”

### `实际结果`

- 记录真实现象或代码证据
- 可写错误日志、字段缺失、条件分支遗漏、刷新链路断裂等
- 不能只写“有问题”或“感觉不对”

### `根因分析`

- 收集阶段若尚不明确，可写 `待 step5 分析`
- 如果通过审查已能锁定大概率根因，可写“初步判断”，但不要假装已经完成修复分析

### `修复方案`

- 收集阶段若尚未进入修复，可写 `待 step5 评估`

### `回归结果`

- 初次收集默认：
  - `回归状态：PENDING`
  - `回归说明：待进入 step5 修复后验证`

## 增量更新规则

对已有 `bug.md` 重新执行问题收集时：

1. 先读取旧文档，不要整份覆盖
2. 已存在的 `BUG-*` 保留编号和历史信息
3. 本轮只补新来源、新现象和新的优先级判断
4. 新问题新增条目，不要塞进旧 Bug 里混写
5. 顶部 `修复进度`、`本轮来源`、`优先处理` 必须同步更新

## 常见错误

| 错误做法 | 正确做法 |
|---------|---------|
| 只在聊天里说“这里有几个问题” | 全部写入 `bugDocs/bug.md` |
| 来自 `bugDocSources` 的问题不记录来源文件 | 在 `来源文件` 写真实路径 |
| 审查发现的问题只写“可能有风险” | 写清违反了哪条预期、哪段代码有证据 |
| 收集阶段就把状态写成 `FIXING` 或 `FIXED` | 先保持 `OPEN` 或 `BLOCKED` |
| 新旧问题混成一个大条目 | 独立问题独立编号 |
