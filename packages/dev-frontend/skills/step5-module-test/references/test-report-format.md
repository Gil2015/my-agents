# 测试报告格式

`.ai/test-reports/{module}.report.md` 的模板。

## 完整模板

```markdown
# Test Report: {module-name}

## Date: {YYYY-MM-DD}
## Tester: AI + {developer name}
## Module: src/modules/{ModuleName}/

## Summary

| 指标 | 值 |
|--------|-------|
| 验收标准总数 | {X} |
| 通过 | {Y} |
| 失败 | {Z} |
| 阻塞 | {W} |
| 通过率 | {Y/X * 100}% |
| 自动化测试数 | {A} |
| 手动检查数 | {B} |

---

### REQ-001: {需求标题}

| 验收标准 | 描述 | 测试类型 | 结果 | 依据 | Bug ID |
|----|-------------|-----------|--------|----------|--------|
| AC-001 | {验收标准文本} | 自动 | PASS | Test: `describe > it "AC-001"` | — |
| AC-002 | {验收标准文本} | 手动 | FAIL | 步骤: 1. 打开页面 2. 点击 X → 显示错误 | BUG-001 |
| AC-003 | {验收标准文本} | 自动 | PASS | Test: `describe > it "AC-003"` | — |
| AC-004 | {验收标准文本} | 手动 | BLOCKED | 接口尚未部署 | — |

---

### REQ-002: {需求标题}

| 验收标准 | 描述 | 测试类型 | 结果 | 依据 | Bug ID |
|----|-------------|-----------|--------|----------|--------|
| ... | ... | ... | ... | ... | ... |

---

## 边界用例测试

| 类别 | 测试项 | 结果 | 备注 |
|----------|------|--------|-------|
| 空数据 | 列表 0 条记录 | PASS | 显示空状态占位图 |
| 大数据量 | 加载 500 条数据 | PASS | 分页正常，无卡顿 |
| 特殊字符 | 名称含 emoji 🎉 | FAIL | 表格中 emoji 被截断 | BUG-003 |
| 空值字段 | updatedAt = null | PASS | 显示"—"兜底 |
| 网络异常 | 接口返回 500 | PASS | 显示错误提示信息 |

---

## Bug 汇总

| Bug ID | 关联验收标准 | 严重程度 | 描述 | 建议修复方案 |
|--------|-----------|----------|-------------|---------------|
| BUG-001 | REQ-001/AC-002 | 高 | 提交时未触发表单校验 | 提交前添加 form.validateFields() |
| BUG-002 | REQ-002/AC-001 | 中 | 日期显示 UTC 而非本地时间 | 使用 dayjs.format() 并设置时区 |
| BUG-003 | 边界/特殊字符 | 低 | 表格列中 emoji 字符被截断 | 增加列宽或添加省略号 |

---

## 自动化测试结果

```
Test Suites: 1 passed, 1 total
Tests:       {X} passed, {Y} failed, {Z} total
Time:        {N}s
```

## 备注

- {任何观察、建议或上下文信息}
- {已知限制或延后处理的事项}
```

## 结果状态定义

| 状态 | 含义 | 所需操作 |
|--------|---------|----------------|
| PASS | 测试通过，满足验收标准 | 无 |
| FAIL | 测试失败，不满足验收标准 | 创建 Bug 记录，必须修复 |
| BLOCKED | 因外部依赖无法测试 | 记录阻塞原因，后续再测 |

## 严重程度定义

| 严重程度 | 含义 | 示例 |
|----------|---------|---------|
| 致命 | 功能完全不可用，有数据丢失风险 | 保存按钮导致数据被删除 |
| 高 | 核心功能受损 | 表单校验被绕过 |
| 中 | 功能可用但存在问题 | 日期格式错误、轻微 UI 异常 |
| 低 | 外观或轻微可用性问题 | 对齐偏差 2px、文字错别字 |

## Bug 列表格式

对于开发者标记由 AI 修复的 Bug，写入 `.ai/test-reports/{module}.bugs.md`：

```markdown
# Bug List: {module-name}

## Date: {YYYY-MM-DD}

### BUG-001: {Bug 标题}
- **关联验收标准:** REQ-001/AC-002
- **严重程度:** 高
- **状态:** OPEN | FIXING | FIXED | WONT_FIX
- **描述:** {出了什么问题}
- **复现步骤:**
  1. {步骤 1}
  2. {步骤 2}
  3. 预期结果: {应该发生什么}
  4. 实际结果: {实际发生了什么}
- **疑似原因:** {Bug 可能出在哪里}
- **建议修复方案:** {如何修复}
- **涉及文件:** {源文件列表}
- **修复指派:** AI | 开发者
```
