# 审查检查清单

对以下**每一项**进行检查，评分为 PASS（通过） / WARN（警告） / FAIL（不合格）。

## A. YAML Frontmatter

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| F1 | 包含 `name` 和 `description` 字段 | 缺少任一字段 |
| F2 | frontmatter 总计 < 1024 字符 | 超出限制 |
| F3 | `name` 仅使用字母、数字、连字符 | 包含空格、下划线或特殊字符 |
| F4 | `description` 清晰描述技能做什么和何时使用 | 缺少功能概要或触发条件，导致触发不准确 |
| F5 | `description` 不包含详细工作流步骤 | 包含具体流程细节（功能概要是可以的） |

补充说明：

- `description` 的核心要求是清晰，不是必须英文。
- 在 `@gai/basic` 包内，除 `name`、代码、命令、路径、字段名、协议字面量和理论上必须英文的内容外，其余文案尽量中文。
- 如果目标仓库已有更具体的语言规范，优先遵循目标仓库规范。

## B. 结构与内容

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| S1 | 有概述部分（1-2 句话） | 缺失或过于冗长（>3 句） |
| S2 | 有"何时使用"且列出具体场景 | 缺失或含糊不清 |
| S3 | 有"不适用场景"或例外说明 | 完全缺失 |
| S4 | 核心流程/内容完整 | 残留 TODO 占位符，或章节为空 |
| S5 | 有快速参考或总结表格 | 没有可快速扫描的参考内容 |
| S6 | 有常见错误或陷阱部分 | 缺失 |

## C. Token 效率

| # | 检查项 | WARN 条件 |
|---|--------|----------|
| T1 | 入口/高频技能尽量精简（200 词以内为佳） | 明显冗长且有可删减空间 |
| T2 | 普通技能 SKILL.md 控制在合理范围（参考 < 500 行） | 明显冗长且有可删减空间 |
| T3 | 各章节间无重复信息 | 内容重复 |
| T4 | 大段参考材料已拆分到 `references/` | SKILL.md 中参考内容超过 100 行 |
| T5 | 使用交叉引用而非复制其他技能内容 | 从其他技能复制粘贴 |
| T6 | 执行流程避免无界上下文扩张 | 默认要求读取整个目录/全部 references/所有匹配结果，或生成超长完整输出，且没有“先缩小范围再深入”的分阶段策略 |

## D. 类型专项检查

根据步骤 1 识别的技能类型，选择对应的附加检查。如果技能不属于以下任何类型，跳过此节。

### 纪律执行型

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| D1 | 有铁律（一句话，代码块格式） | 缺失或含糊 |
| D2 | 有违反后果 | 未定义惩罚 |
| D3 | 合理化借口表 >= 5 条 | 太少或缺失 |
| D4 | 红旗清单 >= 5 条 | 太少或缺失 |
| D5 | 包含"违反字面意思就是违反精神"的防线 | 缺失反"精神 vs 字面"辩护 |
| D6 | 有强制验证步骤（含命令） | 无验证环节 |

### 技术型

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| K1 | 有 before/after 代码示例 | 缺少对比 |
| K2 | 代码示例是领域特定的（非通用 TODO） | 仍有占位代码 |
| K3 | 有常见错误表 | 缺失 |

### 路由型

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| R1 | 有路由表映射任务到子技能 | 缺失 |
| R2 | 有决策流程图 | 缺失 |
| R3 | 覆盖"无匹配"兜底情况 | 缺失兜底 |

### 搜索驱动型

| # | 检查项 | FAIL 条件 |
|---|--------|----------|
| X1 | 搜索命令有文档和示例 | 缺失 |
| X2 | 数据文件（CSV）存在且有表头 | 缺失或格式错误 |
| X3 | 脚本可运行（解释器和路径正确） | 引用失效 |

## E. 质量与清晰度

| # | 检查项 | WARN 条件 |
|---|--------|----------|
| Q1 | 流程图仅用于非显而易见的决策 | 为线性步骤画流程图 |
| Q2 | 密集信息使用表格呈现 | 适合表格的内容用了长段落 |
| Q3 | 无残留 TODO/占位文本 | 任何 "TODO" 残留 |
| Q4 | 格式一致（标题、列表、代码块） | 风格混杂 |
| Q5 | 可操作 — Claude 能无歧义地执行 | 模糊指令如"审查代码质量" |
| Q6 | 未过度使用 MUST/ALWAYS/NEVER 大写强制词 | 满篇强制词而缺乏"为什么"的解释（非纪律执行型技能） |
