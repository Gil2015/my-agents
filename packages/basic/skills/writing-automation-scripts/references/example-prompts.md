# 示例提示词

下面这些说法都适合触发这个 skill，可以直接用，也可以改写。

## 最简触发

```text
帮我写一套本地启动项目脚本。
```

```text
帮我在 scripts 里做一套支持传参的自动化脚本。
```

```text
帮我写一个电脑操作脚本，把我每天重复做的本地流程自动化。
```

## 多项目晨起启动

```text
我每天开机后要打开 IDE，切到 4 个子项目执行 npm link 和 npm run build:watch，
再切到主项目执行 npm link xxx xxx 和 npm run dev，最后打开浏览器。
请给我一套脚本目录，默认串行执行，必要时再后台化。
```

## 拆分多文件交付

```text
给我拆成多个脚本文件放在 scripts/yq-morning-boot 里，最好保留 steps 子目录，方便我后面单独调试。
生成完后请附带 README.md，说明每个脚本怎么用。
```

## 单入口交付

```text
给我一个 run.command，内部串起来执行所有步骤；如果还需要辅助脚本，也一起放进同目录。
如果最终只交付一个脚本文件，请额外生成同名 md 使用说明。
```

## 电脑操作

```text
帮我写一套 macOS 自动化脚本：打开 IDE、打开浏览器指定地址、整理下载目录，并弹出完成提示。
如果某一步更适合 AppleScript，你自己选。
```

```text
给我一个脚本，把桌面上的截图按日期移动到指定文件夹，并在执行前支持 dry-run。
脚本里请加注释，我后面要看得懂。
```

## 参数化

```text
写一个本地自动化脚本，参数支持 --workspace、--mode、--open-browser。
如果没传就走默认值，并把使用示例写进说明文档。
```

## AI 调用

```text
帮我写一个本地脚本，读取剪贴板内容，调用我已经装好的 AI CLI 做总结，然后把结果写到桌面文件里。
如果你需要假设依赖，请明确写出来。
```

## 对就绪判断敏感

```text
watch 命令需要先彻底起来，再执行下一个步骤。请你自己判断用 shell 还是 node 辅助脚本更合适，
并把等待逻辑写稳一点。
```

## 对风险敏感

```text
脚本里如果要 kill 旧进程、删临时目录、或者改 link，请显式提醒风险，不要静默生成危险命令。
```
